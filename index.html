How the App Works

NEWS-e Forecast Viewer is a desktop web application that allows for the easy selection of image products to view by selecting important parts of image file names from menus. There are images corresponding to day (Date), time (Forecast, Minute, Hour, Period), Percentile, and product (Environment, Rad/Sat/QPF, Severe, Rotation, Verification, Members). A choice from any of the menus will update the src value of the <img> element holding the image product in the middle of the view. Menus rely on metadata provided in a json file that should be named for the build and placed in public/data relative to src/. The Warn-On Forecast group uses NEWS-e Forecast Viewer to conduct user research studies on forecasters.

For the Dates menu to be accurate, when the app is in active use, public/data/getdates.sh must be run (on the server where the build is live) every day until the user tests are complete.

For Developers

NEWS-e Forecast Viewer has been built out several times for different user groups. Each build has its own branch in git. Typing

git branch

in console while in the directory will reveal branches for builds. master refers to "realtime". Differences in builds include changing the survey link and other minor layout differences, but generally revolve around the name of the json file referenced in the main app container to fetch. the json data structure is very specific. The Warn-On Forecast team knows this data structure and is responsible for providing the file to the app for each build.

Debugging the App

This project was bootstrapped with Create React App. Check out the most recent documentation for CRA here. To truly feel comfortable with the codebase, you will need to be familiar with CRA, and React.js in particular. The documentation for React is very robust and relatively easy to read and understand if you understand Javascript and web development principles. Starting there if you are new to React will be a big help. Otherwise, reading through this document (where you are now) will help you understand the code as I have written it.

To the left the components that make this app work are organized and documented based on the way the src folder is organized with respect to .js files. Every class, function, and state object has been commented.

Useful Tools

If the app has issues, a helpful tool to debugging is the React Developer Tools browser extension for Chrome and Firefox.

React syntax is very well supported in Atom text editor, a free text editor with many useful packages for developers. It is available in the Self Service app store.

Reproducing the App

From the terminal, navigate to the directory where the src folder is. For example:

cd rwof

Make any changes that may need to be made in src, and again from terminal:

npm run build

This command will produce a production build of the app in vanilla webstack code. The contents of the build folder that appears level with src can be moved to a live web server.

Important Little Things

A production build will not work properly if the package.json file lacks an attribute for the homepage. This information gets translated to complete relative links written in "syntactic sugar" or short hand code.

So before running the build command, be sure to add the relative live address for where this app will live on the web server to package.json on the outermost attribute level on the tree. For example:

"homepage": "/projects/wof/news-e/realtime"

is appropriate for a build that has its index.html placed at https://www.nssl.noaa.gov/projects/wof/news-e/realtime.

In this example, build was renamed to realtime.
